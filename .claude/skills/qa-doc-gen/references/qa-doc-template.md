# QA 测试文档模板

## 文档头部

```markdown
# {模块名称} - {测试范围描述}

**模块**: {模块名称}
**测试范围**: {测试范围简述}
**场景数**: {N}
**优先级**: 高 / 中 / 低
```

## 可选：背景说明

仅在功能涉及 API 端点或非 UI 交互时添加：

```markdown
---

## 背景说明

{功能背景描述}

端点：`METHOD /api/v1/...`

请求/响应示例（如适用）
```

## 可选：数据库表结构参考

仅在需要数据库验证时添加：

```markdown
---

## 数据库表结构参考

### {table_name} 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | CHAR(36) | UUID 主键 |
| ... | ... | ... |
```

## 场景结构（每个场景）

```markdown
---

## 场景 {N}：{场景标题}

### 初始状态
- {前置条件 1}
- {前置条件 2}

### 目的
{验证的功能点}

### 测试操作流程
1. {步骤 1}
2. {步骤 2}
   - 子步骤（如填写表单字段）
3. {步骤 3}

### 预期结果
- {UI 预期表现 1}
- {UI 预期表现 2}

### 预期数据状态
```sql
{验证 SQL}
-- 预期: {期望值}
```
```

## 通用场景（可选）

文档末尾可添加一个通用场景，不计入编号。适用于认证检查、通用错误处理等。

```markdown
---

## 通用场景：{标题}

### 测试操作流程
1. ...

### 预期结果
- ...
```

## 检查清单

```markdown
---

## 检查清单

| # | 场景 | 状态 | 测试日期 | 测试人员 | 备注 |
|---|------|------|----------|----------|------|
| 1 | {场景1标题} | ☐ | | | |
| 2 | {场景2标题} | ☐ | | | |
| ... | ... | ☐ | | | |
```

## 关键规则

1. **每个文档不超过 5 个场景**（通用场景不计入），便于并行测试
2. **场景需覆盖**：正常流程 + 异常流程 + 边界条件
3. **预期数据状态**中的 SQL 使用 `{placeholder}` 表示动态值
4. **UI 操作描述**使用中文书名号「」标注按钮/菜单名称
5. **API 测试**提供完整 curl 命令示例
6. **所有文字使用中文**，技术术语（SQL、API 路径等）保持英文
7. **UI 可见性**：涉及 UI 的功能追加/更新/重构时，必须包含入口可见性场景（验证导航链接、图标、计数等），后续操作场景的 Portal UI 流程应从用户可见入口导航进入，禁止仅通过直接输入 URL 测试
