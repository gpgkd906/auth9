# UI/UX 测试 - Dashboard 概览页

**模块**: 页面专项
**测试范围**: 统计卡片布局、最近动态列表、操作按钮响应式、文字间距
**场景数**: 5

---

## 场景 1：统计卡片按钮与交互响应式

### 目的
验证统计区域在不同宽度下，按钮和点击区域的尺寸能够自动调整，且同一行组件高度保持严格一致。

### 测试操作流程
1. 访问 `/dashboard`。
2. 观察统计卡片右上角的“查看详情”或“时间筛选”按钮（如有）。
3. 调整窗口宽度从 1280px 到 375px。
4. 使用测量工具检查同一排卡片内，不同内容长度下的总高度。

### 预期视觉效果
- **Desktop**: 按钮保持紧凑尺寸，靠右对齐。
- **Mobile**: 按钮宽度自动扩展至 100%，增加点击热区（至少 44px 高）。
- **高度对齐**: 同一排的所有卡片即使文字行数不同，高度也必须强制相等（`h-full`）。
- **间距**: 按钮与上方数值之间的距离在移动端应从 12px 增加到 16px 以防误触。

---

## 2. 标题区域文字下边距与图标对齐

### 目的
验证 Dashboard 主标题与其下方描述文字、左侧图标（如有）的垂直居中关系。

### 预期视觉效果
- **垂直居中**: 如果标题旁有图标或 Badge，它们的中心轴线必须与文字的 X-Height 中心精确对齐。
- **间距**: `h1` 与描述文字之间应为 `mb-1` (4px)。
- **分区间距**: 描述文字与下方统计网格之间应为 `mb-6` (24px)。
- **视觉流**: 标题区域应有明显的留白，避免与卡片粘连。

---

## 3. 最近动态 (Recent Activity) 列表对齐

### 目的
验证动态列表项在不同字数下的排版稳定性。

### 测试操作流程
1. 查看最近动态列表。
2. 检查事件名称（如 `tenant.update`）与时间戳之间的对齐。

### 预期视觉效果
- **文字对齐**: 事件名称加粗，颜色为 `--text-primary`；时间戳位于下方，颜色为 `--text-tertiary`。
- **垂直居中**: 左侧的小圆点（Activity Dot）应与第一行文字精确对齐。
- **边距**: 每行动态之间的 `py-3` 应保持视觉上的呼吸感。

---

## 4. 窄屏下的空状态 (Empty State) 视觉

### 目的
验证当没有数据时（如 0 个租户），引导按钮的响应式表现。

### 测试操作流程
1. 在无数据状态下访问 Dashboard。
2. 观察中间的引导卡片和“开始创建”按钮。

### 预期视觉效果
- **按钮尺寸**: 在 375px 下，按钮应变为大号（Large）并横向撑满。
- **文字换行**: 描述文字不应在窄屏下出现单字成行的情况。

---

## 5. 玻璃卡片阴影一致性检查

### 目的
验证 Dashboard 页面所有卡片（Stat vs Activity）的阴影层级。

### 测试操作流程
1. 观察页面上所有 `.liquid-glass` 元素。
2. 检查悬停前后的 `box-shadow`。

### 预期视觉效果
- **层级**: 统计卡片作为一级信息，阴影应比最近动态列表稍微深一点（`shadow-lg`）。
- **高光**: 顶部 1px 的内发光线在暗模式下应清晰可见，增加质感。
